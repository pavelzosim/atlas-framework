Procedural Tire Generator – Scaling and Physical Consistency

Objective

The goal of this tool is to generate a flexible and physically accurate tire model based on tire measurements and a specific tread pattern. It accounts for both symmetrical and asymmetrical tread designs.

Key Parameters

To ensure the generator behaves like a real-world object, the system uses inputs derived from standard tire specifications:

  • Tire Width (mm): The width of the tire from sidewall to sidewall.
  • Aspect Ratio (%): The ratio of the tire's sidewall height to its width.
  • Wheel Diameter (inches): The diameter of the wheel on which the tire is mounted.

System Logic

The internal logic creates a cascading dependency where the base parameters define the final geometry:

  • Tire Width acts as the master scale for the tread and sidewall surface.
  • Aspect Ratio drives the vertical displacement of the sidewall.
  • Wheel Diameter dictates the central aperture size.
Dynamic scaling based on real-world inputs

Technical Implementation (VEX)

The core of the system relies on strict unit conversion. Since Houdini operates primarily in meters, normalizing diverse units (mm, inches, %) is critical to maintaining a predictable scale. 

// Input parameters
float tireWidth = chf('tire_width'); // mm
float aspectRatio = chf('aspect_ratio') * 0.01; // % to decimal
float wheelDiameter = chf('wheel_diameter'); // inches

// Unit normalization to meters
f@tireWidth = tireWidth / 1000; // mm to meters
f@sidewallHeight = (tireWidth * 0.03937 * aspectRatio) / 37.37; // sidewall height in meters
f@wheelDiameter = wheelDiameter / 39.37; // inches to meters

Pattern Tiling and Adjustments

Once the tire's profile is defined, the tread pattern is tiled. To ensure proper meshing, the minimum division for the tread sections is set to 32. The tile itself is represented by a curve spanning from the outer circle to the inner circle of the tire.

To avoid point overlaps or misalignments along the length of the tile, each section is adjusted by calculating the angle (θ) for every spline section point. These corrections ensure a seamless, closed-loop cylinder without the need for manual vertex tweaking.

Preparing for Extrusion

After generating the base tire shape, the geometry requires vertical and horizontal division for controlled extrusion. Standard Houdini tools proved inadequate here due to variations in polygon sizes and orientations, which led to the development of a custom tool.

This custom approach ensures uniform splitting by:

  • Calculating edge weights and iteratively dividing groups.
  • Maintaining consistent polygon edge lengths across the topology.
  • Preserving circular radii to prevent geometric distortion during processing.

Radial Polygon Displacement

For the final tread height and shape, a specific displacement tool is used. It allows for the grouping of edge loops, supporting both symmetrical and asymmetrical patterns. By implementing soft selection, we ensure smooth transitions between displaced areas, providing high precision and flexibility in design.

UV Mapping Strategy

To support both complex tread patterns and sidewall branding, a dual UV mapping approach is employed:

1. Pattern-Based UVs

Divides the tire into distinct quadrants. This allows for dedicated mapping of the tread and sidewall areas, essential for high-fidelity texturing.

2. Cylindrical UVs

Ensures seamless wrapping along the tire's circular profile, vital for minimizing seams in game engines.

Sidewall Markings & SVG Export

To allow for detailed sidewall customization, I built an SVG export tool. It converts geometry into splines and generates editable vector tiles. These can be edited in external software and re-applied to the tire model.

The utility of this tool extends beyond just tire branding; it functions as a general-purpose geometry-to-vector converter, making it highly reusable for other hard-surface assets.

Procedural geometry to SVG conversion workflow

The core logic handles coordinate space transformation and path data generation: 

# Define SVG dimensions for proper scaling
width = maxsize if size.x() > size.y() else maxsize * size.x() / size.y()
height = maxsize * size.y() / size.x() if size.x() > size.y() else maxsize

# Transform points from Houdini geometry to SVG coordinates
def transform_points(points):
    return [hou.Vector2(
        (p.x() - minv.x()) / size.x() * width,
        (1.0 - (p.y() - minv.y()) / size.y()) * height
    ) for p in points]

# Write SVG path data to file
with open(filename, 'w') as fp:
    fp.write('<svg width="{}px" height="{}px" version="1.1">\n'.format(width, height))
    for prim in geo.iterPrims():
        if prim.type() == hou.primType.Polygon:
            points = transform_points([v.point().position() for v in prim.vertices()])
            fp.write('<path d="M{} {}{}"/>\n'.format(
                points[0].x(), points[0].y(),
                ' '.join(f'L{p.x()} {p.y()}' for p in points[1:])
            ))
    fp.write('</svg>')

Tread Pattern Creation

The tread pattern workflow is designed to balance creative freedom with procedural stability. Artists can utilize standard modeling operations — such as extrusion, deletion, and displacement — while the system handles the complex backend math.

Automated calculations ensure the following:

  • Seamless Tiling: Ensures the pattern loops perfectly around the tire's circumference without artifacts.
  • Height Adjustment: Automatically calculates and maintains uniform depth for extruded areas, regardless of tire size.
  • Surface Alignment: Automatically conforms the pattern to the curved surface of the tire, eliminating the need for manual corrections or vertex snapping.

By abstracting the technical "heavy lifting" (math, snapping, and looping), the artist can focus purely on design, knowing that the procedural consistency of the model will remain intact.
