Отлично! Рад, что всё завелось. Да, CDN-кэш — штука коварная: если он однажды запомнил пустой ответ или ошибку под этим тегом, он будет отдавать её до победного. Стратегия со сменой версии (даже минорной или патчевой, вроде `v1.0.4`) — самый надёжный способ пробить этот затык.

Я сгруппировал их по модулям с именами CSS-классов и кратким описанием назначения, чтобы ты мог использовать этот список как быстрый индекс для навигации по коду.

### 📦 ИНВЕНТАРНАЯ КАРТА ATLAS FRAMEWORK (Текущий статус)

#### **MODULE 1: Базовая типографика и вертикальные ритмы**

* `h2` — Главный ориентир верхнего уровня (авто-шеврон `»`, CAPS, жесткий отступ сверху `36px` для отделения от предыдущих блоков).
* `h3` — Системный подблок (авто-слэши `//`, подчёркивание dashed-линией, CAPS).
* `h4` — Локальная спецификация (авто-тире `—`, обычный регистр для читаемости, плотный маржин без линий).
* `p` — Параграф текста (`line-height: 1.7`, моноширинный шрифт, аккуратный сброс маржина для первого элемента в теле статьи через `#atlas-body > *:first-child`).
* `strong` / `em` — Акценты (максимальный контраст для ключевых токенов и приглушенный наклонный стиль для имитации комментариев в коде).

#### **MODULE 2: Терминальные списки и инлайн-токены**

* `ul` + `li` — Маркированный список в стиле CLI (двухслойный маркер: жесткие скобки `[ ]` через `::before` и абсолютный пунцовый шеврон `>` через `::after` с защитой от излома строки `white-space: nowrap`).
* `ol` + `li` — Нумерованный список (автоматический лог-счетчик в формате `[ 01 ]`, `[ 02 ]` с ведущим нулём через `decimal-leading-zero`).
* `code:not(pre code)` — Инлайн-токены (компактные системные плашки для переменных и типов данных с тонкой рамкой и легким фоном).

#### **MODULE 3: Система технического сравнения (Side-by-Side)**

* `.atlas-compare` — Двухколоночный контейнер с "прицелами" (плюсами `+`) на углах рамки. Автоматически схлопывается в одну колонку на мобильных экранах (`max-width: 680px`).
* `.atlas-compare__column` — Колонка (левая/правая).
* `.atlas-compare__badge` — Верхний технический шильдик (например, `BEFORE / URP` или `AFTER / COMPUTE SHADER`).
* `.atlas-compare__content` — Изолированная контентная зона с поддержкой `.atlas-compare__concept` (заголовок фичи) и внутренних параграфов.

#### **MODULE 4: Разделители контекста**

* `.atlas-section-break[data-index="0X"]` — Глобальный сквозной разделитель секций сплошной линией. Номер секции пробрасывается динамически через HTML-атрибут `data-index` и центрируется поверх линии.
* `.atlas-eof-divider` — Концевой маркер статьи (`[ EOF ]` / `End of File`) с автоматическим заполнением пространства точечной линией `dashed` по бокам.

#### **MODULE 5: Паспорт статьи (Blueprint / Метаданные)**

* `.passport-bp` — Главный монолитный блок карточки статьи с жесткой двойной рамкой (`2px solid`).
* `.bp-title-bar` — Эмуляция командной строки в шапке (путь к файлу, статус системы, имя пользователя).
* `.bp-main-title` — Крупный заголовок лога/статьи.
* `.bp-grid` + `.bp-cell` — Двухколоночная матрица ключевых параметров (справа/слева) с фиксированным выравниванием ключей `.bp-key`.
* `.bp-meta` — Строка системных флагов (Дата, Чтение, Секция).
* `.atlas-blueprint-link` — Интерактивные теги-кнопки в стиле интерфейса **Volkov Commander** (при ховере намертво заливаются DOS-синим цветом `#0000aa` и отбрасывают дизеринг-тень `2x2px` через conic-gradient).

#### **MODULE 6: Подвал сайта (System Footer)**

* `.atlas-footer-wrapper` — Ограничивающий контейнер подвала.
* `.atlas-footer-block--main` — Контурный блок футера с двойной рамкой.
* `.atlas-footer-block__support-header` — Шапка подвала с текстовым Kaomoji (японским смайлом) и статусом.
* `.atlas-footer-block__matrix-zone` — Трехколоночная сетка на десктопе, которая перестраивается в один стек на мобильных для ссылок на категории, архивы и контакты.

#### **MODULE 7: Таблицы данных и системные метрики**

* `.atlas-table-responsive` — Контейнер с горизонтальным скроллом, защищающий широкие таблицы от вылезания за границы мобильного экрана.
* `.atlas-table` — Нативная таблица с collapse-границами, принудительно защищенная от Wix-сбросов стилей.
* `.atlas-table-num` — Класс для выравнивания ячеек с числами по правому краю с фиксацией ширины цифр (`tabular-nums`), чтобы значения в столбцах профайлера не "плясали".
* `.atlas-param[data-status="ok/warn/crit"]` — Цветовые статусы метрик (зеленый/желтый/красный) для подцветки загрузки GPU, пропускной способности памяти или критических ошибок.
