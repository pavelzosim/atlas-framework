#### **Файл: `cdn/atlas-typography.css**`

**MODULE 1: HEADINGS & DOCUMENT RHYTHM**

* `h2` — Главный ориентир верхнего уровня: CAPS-заголовок с усиленным межбуквием и увеличенным верхним отступом для разрыва секций.
* `h2::before` — Префикс `»` перед `h2` для мгновенной визуальной навигации по крупным блокам.
* `h3` — Системный подблок: CAPS, компактнее `h2`, с dashed-подчёркиванием для технического ритма.
* `h3::before` — Префикс `//` перед `h3`, имитирующий комментарий/служебную метку.
* `h4` — Локальная спецификация: плотный подзаголовок в обычном регистре без разделительной линии.
* `h4::before` — Префикс `—` для визуального отделения локальных пунктов.

**MODULE 2: BODY TEXT & INLINE SEMANTICS**

* `p` — Базовый параграф текста с выверенным межстрочным интервалом для чтения длинных технических логов и статей.
* `strong` — Полужирное выделение ключевых терминов и флагов компилятора в том же семействе шрифта.
* `em` — Лаконичный курсив для примечаний, исключений или цитирования спецификаций.
* `kbd` — Стилизация аппаратных клавиш и клавиатурных сочетаний (моноширинный шрифт, рамка, легкий сдвиг вниз для эффекта объема).
* `blockquote` — Выделенный блок цитаты сплошным левым бортом (border-left) для акцентирования внимания на внешних источниках или документации.

**MODULE 3: INLINE CODE ARCHITECTURE**

* `code:not(pre code)` — Инлайн-код для упоминания переменных, функций и типов данных прямо в теле параграфа. Изолирован от Prism.js, чтобы исключить графические конфликты с крупными блоками кода.

#### **Файл: `cdn/atlas-layouts.css**`

**MODULE 1: SECTION BREAKS & NAVIGATION INDICES**

* `hr.atlas-section-break` — Графическая тонкая линия-разделитель, пробивающая контент по всей ширине.
* `hr.atlas-section-break[data-index]::after` — Автоматический вывод индекса секции в формате `[ SECTION_INDEX // X ]` по правому краю, перекрывающий линию подложкой.

**MODULE 2: MEDIA CONTAINERS & INSPECTOR LOGS**

* `.atlas-media-node` — Монолитный изолированный контейнер для скриншотов, схем или рендер-тестов.
* `.atlas-media-node__wrapper` — Темная подложка (имитация темы Unity/Unreal Inspector) для центрирования графики.
* `.atlas-media-node img` — Адаптивное изображение с ограничением максимальной ширины и защитой от излома верстки.
* `.atlas-media-node__caption` — Техническая подпись под медиафайлом, стилизованная под консольный вывод (лог) с префиксом `[MEDIA_LOG // FILE_ATLAS...]`.

#### **Файл: `cdn/atlas-schema.css**`

**MODULE 1: MATRIX DATA TABLES**

* `.atlas-table-responsive` — Контейнер с горизонтальным скроллом, защищающий широкие таблицы от обрезания и излома на мобильных экранах Wix.
* `.atlas-table` — Системная таблица для сравнения сущностей, типов данных или параметров GPU.
* `.atlas-table th` — Шапка таблицы: CAPS, мелкий шрифт, увеличенный трекинг, фоновая подложка в стиле отчетов профайлера Unity.
* `.atlas-table td` — Ячейки данных с фиксированным вертикальным ритмом.
* `.atlas-table-num` — Принудительное выравнивание чисел по правому краю с активацией `tabular-nums` для предотвращения горизонтального дрожания цифр при прокрутке.

**MODULE 2: METRIC PARAMETER STATUSES**

* `.atlas-param` — Изолированный компактный инлайн-бейдж статуса параметра внутри ячеек таблиц.
* `.atlas-param[data-status="ok"]` — Зеленый маркер-индикатор корректного или оптимизированного состояния.
* `.atlas-param[data-status="warn"]` — Желтый маркер-индикатор потенциальной угрозы производительности (оверхед, лишние сэмплы).
* `.atlas-param[data-status="crit"]` — Красный маркер-индикатор критической ошибки (нагрузка на ALU, отсутствие constant buffer).

**MODULE 3: PIPELINE FLOW LAYOUTS**

* `.atlas-pipeline` — Вертикальный стек (конвейер) для пошаговой визуализации проходов рендеринга или этапов автоматизации.
* `.atlas-pipeline-step` — Элемент шага с жестким левым акцентным бортом.
* `.atlas-pipeline-step strong` — Маркер фазы конвейера (например, `[Vertex Shader Stage]`) с фиксированной минимальной шириной для выравнивания всей структуры по сетке.

#### **Файл: `cdn/atlas-code-theme.css**`

**MODULE 1: CODE BLOCK CONTAINER**

* `pre.atlas-code-block` — Глобальный монолитный контейнер для многострочных сниппетов кода (HLSL, VEX, Python). Включает горизонтальный скролл для длинных строк и защиту от скруглений углов.
* `pre.atlas-code-block code` — Базовые настройки моноширинного текста, высоты строки (line-height) и цвета базового слоя шрифта.

**MODULE 2: SYNTAX HIGHLIGHTING TOKENS (PRISM.JS ENGINE)**

* `.token.comment`, `.token.prolog` — Приглушенный серый цвет для комментариев и служебных заголовков кода.
* `.token.keyword`, `.token.atrule` — Выделение ключевых слов языка программирования (структуры, директивы, прагмы).
* `.token.function` — Акцентный цвет для вызовов функций и методов (`UnityObjectToClipPos`, `tex2D`).
* `.token.string`, `.token.char` — Выделение строковых литералов и путей инклудов файлов.
* `.token.number`, `.token.boolean` — Оформление численных констант и булевых флагов.

#### **Файл: `cdn/atlas-comparison.css**`

**MODULE 1: SIDE-BY-SIDE FLEX MATRIX**

* `.atlas-compare` — Двухколоночный флекс-контейнер для прямого сопоставления технологий (например, BIRP vs URP, Плохо vs Хорошо). Автоматически перестраивается в одну колонку (stacking) на мобильных дисплеях ниже 680px.

* `.atlas-compare__badge` — Полноширокий верхний бэйдж колонки в стиле заголовков DOS-навигаторов для обозначения контекста (например, `// BAD_APPROACH`).
* `.atlas-compare__concept` — Внутренний заголовок конкретной фичи или архитектурного узла внутри колонки сравнения.
